 After the call of swap(jungleDVD, cinderellaDVD), the titles of the two DVDs remain unchanged because Java passes parameters by value. The swap method only swaps the copies of the references, not the original references.
 After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()), the title of jungleDVD changes because the method modifies the object that jungleDVD refers to by using dvd.setTitle(title).
Java is a pass-by-value programming language. When passing objects, Java passes the value of the reference, not the object itself.

In string concatenation inside loops, StringBuilder and StringBuffer are faster than using the + operator because String objects are immutable. Each concatenation creates a new String object, which produces a lot of garbage in memory.
